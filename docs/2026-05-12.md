## 5. 버그 수정: 관리자 댓글 삭제 후 카운트 표시 오류

### 증상
관리자가 댓글을 삭제했는데도 자유게시판 목록에서 댓글 수가 그대로 "1"로 표시.

### 원인
`/api/admin/comments/[id]` (DELETE) 가 `comments` 테이블에서만 삭제하고, `posts.comments_count` 캐시 컬럼을 갱신하지 않음.

### 수정 1 — 관리자 삭제 API
```ts
// 댓글 삭제 전 post_id 조회
// 삭제 후 실제 댓글 수 재계산 → posts.comments_count 동기화
```
파일: `src/app/api/admin/comments/[id]/route.ts`

### 수정 2 — 자유게시판 목록
캐시 컬럼 의존을 없애고 **실시간 조인 카운트** 사용:
```ts
.select('*, profiles(nickname, avatar_url), comments(count)')
```
표시: `post.comments?.[0]?.count ?? 0`
파일: `src/app/community/page.tsx`

→ 기존에 잘못 캐시된 stale 데이터(예: 1)도 즉시 0 으로 정상 표시.
